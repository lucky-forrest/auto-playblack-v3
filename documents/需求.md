## 1. 项目背景

### 1.1 项目简介

开发一个符合B0304_EQP_Socket标准接口文档的Socket客户端程序，作为EAP客户端与测试设备进行Socket通信。程序接收设备发来的操作指令，执行相应的自动化操作（鼠标移动、点击、窗口控制等），并将执行结果反馈回设备。

### 1.2 项目目标

1. **Socket客户端角色**：按照B0304协议与测试设备建立Socket连接
2. **操作指令执行**：接收并执行设备发送的操作指令序列
3. **协议实现**：严格遵循B0304标准接口文档实现通信协议
4. **实时响应**：作为实时控制客户端，异步接收和处理指令
5. **稳定性**：支持连接断线重连、错误恢复等机制
6. **双模式支持**：支持Socket实时模式和CSV回放模式

## 2. 功能需求

### 2.1 Socket客户端核心功能

#### 2.1.1 Socket连接管理

- **基础连接功能**
  - 使用socket模块建立客户端连接
  - 配置IP地址和端口号（默认127.0.0.1:8888）
  - 支持TCP协议的Socket连接
  - 支持连接状态监控

- **连接维护**
  - 支持断线后的自动重连机制
  - 心跳保活机制，防止连接断开
  - 连接状态监控和异常检测

- **协议握手**
  - 实现B0304协议握手流程
  - 协商协议版本（v1.0.3）
  - 建立安全可靠的连接
  - 初始化客户端会话状态

#### 2.1.2 协议通信

- **消息格式遵守**
  - 按照B0304标准实现消息格式规范
  - 消息包含：协议版本、消息类型、序列号、载荷数据、校验信息
  - 实现消息序列化和反序列化
  - 实现校验机制（CRC校验等）

- **消息类型支持**
  - `E_StartTest`: 开始测试指令
  - `E_OperationStart`: 操作开始指令
  - `E_OperationDone`: 操作完成指令（成功）
  - `E_OperationError`: 操作错误指令
  - `E_Ping`: 心跳检测
  - `E_ConnectionLost`: 连接丢失通知

- **消息发送与接收**
  - 异步接收设备发送的指令
  - 将执行结果反馈给设备
  - 消息队列处理（支持多指令并发）
  - 实时响应优先级处理

#### 2.1.3 操作指令执行

接收到的操作指令内容示例：
```json
{
  "cmd_id": 1,
  "timestamp": 1643613456.123456,
  "event_type": "mouse_click",
  "delay": 0.5,
  "x": 725,
  "y": 689,
  "window_title": "Test Software"
}
```

执行要求：
- **mouse_move**: 移动鼠标到指定坐标
- **mouse_click**: 在指定坐标执行左键点击
- **mouse_scroll**: 鼠标滚动操作
- **key_press/key_release**: 模拟键盘按键
- **window_control**: 窗口控制操作（激活、最大化、最小化等）

#### 2.1.4 执行模式

程序支持两种运行模式：

**模式1：Socket实时模式**
- 作为Socket客户端连接设备
- 接收实时操作指令
- 实时执行指令并反馈结果
- 高优先级的实时响应

**模式2：CSV回放模式**
- 读取CSV操作记录文件
- 按照记录的时间顺序精确回放
- 严格遵循时间差进行sleep控制（误差±0.1秒）
- 用于测试和验证参考

### 2.2 CSV文件（仅用于开发参考）

## 注意：CSV文件仅用于开发参考，不是程序执行的资源

**CSV文件信息**：
- **文件路径**：`documents/录制会话_20260402_100410/录制会话_20260402_100410_operation_log.csv`
- **文件格式**：标准CSV格式，UTF-8编码
- **数据性质**：表示人类手动操作过程的记录
- **用途**：开发时的参考，帮助理解设备期望的操作格式

#### CSV包含字段（开发参考）：
- `timestamp`: 时间戳（ISO 8601格式）
- `event_type`: 操作类型（mouse_move, mouse_click, window_change）
- `x, y`: 坐标信息
- `window_title`: 窗口标题
- `control_text`: 控件文本
- 其他窗口和控件属性

#### CSV模式使用说明：
- 当选择CSV回放模式时，程序解析该CSV文件作为操作序列
- 程序自动读取每条记录，控制时间间隔
- 执行鼠标移动和点击操作
- 详细记录执行过程到日志
- 完成后返回执行统计信息

### 2.3 时间控制要求

#### 2.3.1 实时模式时间控制
- 操作执行延迟由设备指令中的`delay`字段指定
- 程序读取指令后立即执行，无需额外等待

#### 2.3.2 CSV回放模式时间控制

1. **时间差计算原理**
   - 获取上一次操作执行的绝对时间戳（`time.time()`）
   - 获取当前操作的录制时间戳（从CSV读取）
   - 计算时间差：`delay = operation_timestamp - current_time`
   - 最小延迟为0秒（避免负数）

2. **执行精度要求**
   - **执行时间误差**：±0.1秒以内
   - **定时机制**：使用Python的`time.sleep()`函数精确等待
   - **时间戳解析**：解析ISO 8601格式的时间字符串
   - **毫秒级处理**：支持毫秒级时间精度

### 2.4 自动化操作实现（开发参考）

使用autoit库实现以下功能：

#### 2.4.1 窗口管理操作

| 操作             | 函数                                    | 用途                   |
| ---------------- | --------------------------------------- | ---------------------- |
| 检测窗口是否存在 | `autoit.win_exists(title)`              | 判断窗口是否已打开     |
| 等待窗口出现     | `autoit.win_wait(title, timeout)`       | 等待指定秒数后窗口出现 |
| 获取窗口句柄     | `autoit.win_get_handle(title)`          | 获取窗口句柄用于激活   |
| 激活窗口         | `autoit.win_activate_by_handle(handle)` | 激活指定窗口           |
| 设置窗口状态     | `autoit.win_set_state(title, state)`    | 最大化/最小化窗口      |
| 获取像素颜色     | `autoit.pixel_get_color(x, y)`          | 检测屏幕元素状态       |

#### 2.4.2 鼠标操作实现

| 操作     | 函数                                             | 说明               |
| -------- | ------------------------------------------------ | ------------------ |
| 鼠标单击 | `autoit.mouse_click("left", x, y)`               | 左键点击指定坐标   |
| 鼠标移动 | `autoit.mouse_move(x, y)`                        | 移动鼠标到指定坐标 |
| 控件点击 | `autoit.control_click(window, control, "text")`  | 点击指定控件       |
| 输入文本 | `autoit.control_set_text(window, control, text)` | 向控件设置文本     |

### 2.5 系统配置要求

#### 2.5.1 Socket配置

```python
# Socket默认配置
DEFAULT_HOST = "127.0.0.1"
DEFAULT_PORT = 8888
SOCKET_TIMEOUT = 30  # 秒
RECONNECT_DELAY = 5  # 秒
```

#### 2.5.2 日志配置

```python
# 日志配置
LOG_LEVEL = "INFO"
LOG_FORMAT = "%(asctime)s - %(name)s - %(levelname)s - %(message)s"
LOG_FILE = "socket_client.log"
```

#### 2.5.3 执行环境要求

- Python 3.6+
- Windows操作系统
- 已安装的测试软件（根据B0304协议要求）
- Python autoit库

## 3. 性能要求

### 3.1 运行指标

- **内存占用**：< 100MB
- **启动时间**：< 5秒
- **响应时间**（Socket模式）：< 100ms
- **执行效率**：无额外等待，严格按照指令时间执行
- **CPU占用**：< 5%（空闲时）

### 3.2 稳定性要求

- 断线后能在30秒内自动重连
- 支持中斢单个操作但不影响其他操作
- 操作失败能记录详细错误信息
- 支持中断和恢复功能（如需要）

## 4. 限制条件

### 4.1 适用范围

1. Socket模式：作为客户端设备连接到Socket服务器
2. CSV模式：仅用于开发参考和回放测试
3. 操作坐标需在目标窗口的活动范围内
4. 需要保持目标软件处于运行状态
5. 窗口状态变化应能适应

### 4.2 不适用场景

1. Socket模式：需要复杂的协议扩展
2. 动态变化的窗口（窗口标题动态变化）
3. 需要深度逻辑判断的操作
4. CSRF/非标准操作类型
5. 需要高并发性能的场景
